
# Enterprise

# USAGE GUIDELINES

- Use for sequential or highlight content such as promotions, destinations, or featured visuals.

- Limit total slides to keep browsing manageable (recommended: up to 6).

- Use image previews when users benefit from visual orientation.

- Use minimal layouts (no preview) when the surrounding UI already provides context.

- Avoid placing essential or time-sensitive information inside a carousel.

- Ensure controls and movement are predictable and non-disruptive.

# BEHAVIOUR (INTERACTIONS)

  
**Image Preview (Enable/Disable)**  
  
- **With previews: **adjacent slides are partially shown to increase orientation and browsing confidence.  
- **Without previews:** provides a cleaner, minimal layout when previews are not needed or space is limited.  
- Previews are informational only and do not act as navigation triggers.  
  
**Auto-scroll (Enable/Disable)**  
  
**Auto-scroll:**  
- Default Behaviour  
- Images advance automatically on a timed interval  
- Automatically pauses on user interaction (hover, swipe, manual navigation).  
- Recommended for promotional or hero content where passive browsing is expected.  
  
**Auto-scroll disabled:**  
- Images move only through user action  
- Recommended for content requiring attention, reading, or precise comparison.  
- Preferred for B2B interfaces or high-detail visuals.  


  

### Navigation
- Users navigate using arrows, dot indicators, or swipe gestures.
- Dots allow direct selection of a specific slide.
- Arrows move one slide at a time forward or backward.

### Looping
- Looping is the default setting. Carousel wraps from last → first and first → last.
- When disabled, arrows become inactive at boundaries.

### Responsiveness
- Control placement adapts to screen width.
- Outer/inner alignment rules remain consistent across breakpoints.
- Images scale proportionally to preserve aspect ratio.
