# USAGE GUIDELINES

- Use Status Chips only to communicate system or service status, not categories, attributes, or filters.

- Status should be instantly understandable through colour and label together.

- Use Pill&hyphen;shaped chips in B2C interfaces (consumer&hyphen;facing apps or websites).

- Do not use purple for status chips, as the colour is reserved for interactive components such as filter chips.Use Rectangular chips in B2B interfaces (admin tools, dashboards, operational systems).

- Icons are optional and should reinforce the status meaning rather than represent the service itself.

- Keep labels short and clear (1–2 words) for quick scanning

- Maintain consistent placement within lists, cards, or tables so status can be easily compared across items.

## Variants

  
**Outline Pill**  
  
- Used commonly in **consumer&hyphen;facing (B2C) interfaces**  
- Provides a lightweight visual indicator suitable for content&hyphen;heavy layouts  
  
**Solid Pill**  
  
- Used when stronger visual emphasis is required in **consumer&hyphen;facing interfaces**.  
  
**Outline Rect**  
  
- Commonly used in **B2B interfaces**, operational tools, dashboards, and admin panels.  
  
**Solid Rect**  
  
- Used in dense enterprise interfaces where status visibility needs to be stronger.  


## States

  
**Grey**  
  
- Represents a **neutral or default status**.  
- Use when no active condition applies or the item is idle or awaiting action.  
  
**Green**  
  
- Represents a **positive, active, or successful status**.  
- Use when a process is running normally, a service is active, or an operation has succeeded.  
  
**Orange**  
  
- Represents a **warning or attention&hyphen;needed status**.  
- Use when a process requires monitoring and a condition may require user action soon.  
  
**Red**  
  
- Represents a **negative or error status**.  
- Use when a process has failed, a service is unavailable, or an error condition has occurred.  
  
**Blue**  
  
- Represents an **informational or transitional status**.  
- Use when a process is in progress, a system state is being updated, or when information needs to be highlighted without implying error.  
  
**Disabled**  
  
- Neutral status with no positive/negative meaning.  
- Used for base/default states.

# CONTENT GUIDELINES

- Labels should be **short and descriptive** (e.g., “Active”, “Error”, “Processing”, “Offline”).

- Use **sentence case**.

- Avoid long phrases inside chips.

- Maintain consistent terminology across the product.
