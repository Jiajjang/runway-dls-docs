# USAGE GUIDELINES

- Use buttons when explicit user interaction is required.

- Lead with an actionable verb that clearly reflects the next step (e.g. Buy Now, Save Flight, Add to Cart).

- Button labels should be concise, with a maximum of 4 words.

- Use Title Case for button labels.

- Avoid using buttons for decorative or non&hyphen;interactive elements.

- Use only one Primary button per screen to maintain clear action hierarchy.

- Button width may either fill the page container or adapt to the text length. The choice is defined by the designer based on project requirements.

- Minimum button width on Desktop is 240px** **to ensure tap target and visual consistency.

## Sizes of the button

  
**Large**  
  
Large buttons are the default for primary and secondary actions in standard layouts, such as *Add to Cart*, *Save*, *Continue*, or *Confirm* on pages and dialogs.  
  
**Medium**  
  
Medium buttons are used alongside input fields, for example *Search*, *Apply*, or *Add* actions within forms.  
  
**Small**  
  
Small buttons are used for tertiary actions, such as an *Edit* button within a paragraph, inline actions in tables, or secondary controls in dense content areas.
