# How to Use the Runway &hyphen; Design Language System (DLS) Documentation

  
**1. Start with Foundations**  
Before using components, review the **Foundations** section to understand:  
- Color tokens  
- Typography  
- Iconography  
- Spacing and radius system  
- Grid and layout rules  
- Image ratio  
- Effects such as shadows, blur and focus rings  
These foundational elements ensure visual consistency across all digital products.  
  
**2. Use Components as Documented**  
When building designs:  
- Use only approved components from the Figma library given to you  
- Do not detach or modify core component structures  
- Follow documented usage guidelines and states (default, hover, active, disabled, etc.)  
- Use design tokens/styles instead of hard&hyphen;coded values  
If a required variation does not exist, you can create a customise components following the foundational guidelines  
  
**3. Follow Usage & Behavior Guidelines**  
Each component includes guidance on:  
- **Usage guidelines:** Explain when and how the component should be used in product interfaces.​  
- **Variants/Types:** Different versions of a component, examples: buttons has a primary, secondary or icon only button​  
- **States:** Different interaction states of a component, examples: buttons have a default state, hover, active, disabled​  
- **Behaviours:** describe how the component functions and responds to user interaction.​  
- **Content guidelines:** for text inside components, examples: Text in the buttons should be short to avoid truncation, or use clear, action&hyphen;oriented verbs​  
Always review these before implementation.  
  
**4. Check for Updates**  
The DLS evolves over time. Before starting new work:  
- Use the latest Figma library version  
- You don't need to update past projects with the new design components. Use the new components only for new pages or projects unless instructed otherwise.  


- Do not redesign existing standardized components

- Do not introduce new foundations styles without approval

- Always design within the system before exploring custom solutions
