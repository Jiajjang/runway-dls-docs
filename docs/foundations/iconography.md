
# Iconography

Iconography is a foundational visual language used 
across CAG digital products to represent actions, 
concepts, content, and system states in a clear 
and consistent way.

---

# USAGE GUIDELINES

The icon system is designed to:

- Improve comprehension and scannability

- Reduce cognitive load in dense or time&hyphen;sensitive interfaces

- Support wayfinding, navigation, and interaction

- Remain consistent across B2C, B2B, and operational products

---

## Feature Icons

  
**Words**  
Feature icons are visually richer icons used to depict or highlight key features and journeys.

They are typically used:  
- As Quick Links  
- Within cards to represent a feature or flow  
- As supporting micro&hyphen;illustrations paired with content  

Feature icons should not be used for:  
- Buttons  
- Standard UI controls  
- System&hyphen;level actions

# SIZE & STYLE RULES

Icons follow a fixed size and style system.

Size constraints:

- Minimum size: 12 × 12 px

- Maximum size: 40 × 40 px


Style by size:

- 24px and below: Solid icon

- Above 24px: Outline icons


Component&hyphen;level usage:

- Buttons with icons use solid icon

- Filter chips use outline icons

---

# ICON CONSTRUCTION

> Some extra info:  
> Always retain the original stroke&hyphen;based version for future edits.

Use the provided Figma icon template when creating icons.

Construction rules:

- Use a **24 × 24 px** base grid

- Use a** 2px stroke width** for outline icons

- Use rounded stroke terminals

- Keep icons simple and literal

- Avoid unnecessary detail
