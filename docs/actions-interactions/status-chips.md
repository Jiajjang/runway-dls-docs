# USAGE GUIDELINES

- Use Status Chips only to communicate system or service status, not categories, attributes, or filters.

- Status should be instantly understandable through colour and label together.

- Use Pill-shaped chips in B2C interfaces (consumer-facing apps or websites).

- Do not use purple for status chips, as the colour is reserved for interactive components such as filter chips.Use Rectangular chips in B2B interfaces (admin tools, dashboards, operational systems).

- Icons are optional and should reinforce the status meaning rather than represent the service itself.

- Keep labels short and clear (1–2 words) for quick scanning

- Maintain consistent placement within lists, cards, or tables so status can be easily compared across items.
