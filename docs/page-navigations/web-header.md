
# Web Header — Admin Header

**Category:** Page Navigations

# USAGE GUIDELINES

- Use at the top of admin or internal management platforms.

- Keep the header consistent across all admin modules.

- Display top-level navigation items in a single horizontal row.

- Keep navigation labels short and high-level (e.g., “Flight Management”, “Maintenance”).

- Place the user account dropdown on the far right for profile-related actions.

- Avoid overcrowding — limit the number of visible navigation items.

- The header should remain fixed at the top when scrolling (recommended for admin tools).

# CONTENT GUIDELINES

- Navigation labels should be short nouns or noun phrases (e.g., “Audit”, “Airline Management”).

- Avoid long labels; aim for 1–2 words per item.
# BEHAVIOUR (INTERACTIONS)

  

### **Navigation Items**
- Clicking an item navigates to the corresponding admin module with the item highlighted.

### **User Account Dropdown**
- Click/tap opens a dropdown menu with: System Switch (optional), User Settings, Logout
- Dropdown closes when clicking outside or selecting an option.
- Text truncation applies if the username is long.

### **Responsiveness**
- On smaller screens, navigation may retain horizontal navigations, or move into a hamburger menu
- User account dropdown remains accessible at all breakpoints.

### **Sticky Behaviour**
- Header typically remains fixed at the top for admin/management tools.
